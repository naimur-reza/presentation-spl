Scope, Storage Class & Lifetime of Variables in C

This presentation covers how variables behave in C — where they are visible, how they are stored, and how long they exist in memory.

Scope

  • Local Scope
    Variables declared inside a function, accessible only within that function
  • Global Scope
    Variables declared outside all functions, accessible throughout the program
  • Block Scope
    Variables declared within { } blocks, accessible only within that block

Storage Classes

  • auto
    Default for local variables, stored in stack, destroyed when function exits
  • static
    Retains value between function calls, exists for entire program duration
  • extern
    Declares variable defined elsewhere, used for global variable sharing
  • register
    Suggests storing in CPU register for faster access (hint to compiler)

Lifetime

  • Local Variable Lifetime
    Created when function is called, destroyed when function returns
  • Global Variable Lifetime
    Created when program starts, destroyed when program terminates
  • Static Lifetime
    Variables with static storage class persist for entire program execution
